# Security Policy

## Supported Versions

Security fixes are handled on the `main` branch. If release branches are added
later, this policy will be updated with the supported version range.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

Preferred options:

- Open a private vulnerability report using GitHub Security Advisories.
- Email `anant@optimism.studio` with the subject prefix `[symphony-ts security]`.

Please include:

- A description of the issue and the affected component.
- Steps to reproduce or a minimal proof of concept.
- The potential impact.
- Any known mitigations.

Please do not open a public GitHub issue or discussion for security reports.

## Response Expectations

Maintainers aim to acknowledge new reports within 3 business days. After
acknowledgement, we will investigate, coordinate a fix when appropriate, and
credit reporters who want attribution.
